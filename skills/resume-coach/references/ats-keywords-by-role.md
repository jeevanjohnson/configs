# ATS Keywords by Role

When tailoring your resume to a job description, identify the keywords most relevant to that role. Mirror this language in your bullets—not as padding, but as genuine description of what you did.

This file organizes common keywords by common entry-level and mid-level roles. Use this as a reference to spot keyword patterns in the job you're targeting.

## How to Use This Reference

1. **Read the job description** and note repeated words and concepts
2. **Turn them into bullets** using the verb suggestions and framework
3. **Don't force it** — only use keywords that genuinely apply to your experience
4. **Mirror phrasing** — if the JD says "scalable architecture," use that phrase instead of a synonym like "robust system"

### Example

**Job Description says:**
> "Design and implement scalable REST APIs. Optimize database queries. Strong in Python and TypeScript."

**Your resume bullet becomes:**
> "**Designed** scalable REST **APIs** with Python, optimizing database queries to reduce latency by 40% and serve 10K+ requests per second"

Maps to: Designed (verb), scalable REST APIs (keywords), Python (keyword), optimized database queries (keyword pattern match)

---

## Anti-Pattern: Keyword Stuffing

❌ **DON'T:**
- List keywords without context: "REST APIs, scalability, TypeScript, microservices"
- Add skills you don't have
- Use a tool name in a bullet where it doesn't belong

✅ **DO:**
- Weave keywords naturally into achievement bullets
- Only claim skills you can speak to in an interview
- Let real accomplishments dictate the language
